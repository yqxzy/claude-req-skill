# /req codebase — Step 3：代码调研

**加载文件**：`_state.md`、`01_changes.md`（只读变更点中的「影响模块」部分）

**执行**：
1. 从变更点提取需要调研的模块列表，列出来让用户确认
2. 逐模块分析（可用 Bash/Read/Grep 工具搜索代码）：
   - 核心类/接口位置
   - 当前数据流
   - 扩展点和限制点
   - 需要注意的历史遗留问题
3. 重点标注：⚠️ 对方案设计有影响的发现 / ✅ 可复用的能力
4. 预览确认后写入 `02_codebase.md`
5. 更新 `_state.md`：Step3 打勾，上次停留更新
