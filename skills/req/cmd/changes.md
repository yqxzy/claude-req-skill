# /req changes — Step 2：变更点

**加载文件**：`_state.md`、`00_background.md`

**执行**：
1. 输出一句话背景摘要，确认上下文
2. 请用户粘贴 PRD / 变更描述（口头描述也可）
3. 整理为四类（有则列，无则省略）：
   - 新增功能：从无到有
   - 修改功能：改变已有行为（注明：原行为 → 新行为）
   - 删除功能：下线/废弃
   - 非功能性：性能、配置、安全等
4. 每条变更标注预计影响的模块
5. 预览确认后写入 `01_changes.md`
6. 更新 `_state.md`：Step2 打勾，上次停留更新
