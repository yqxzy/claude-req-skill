# /req changes — Step 2：变更点

**加载文件**：`_state.md`、`00_background.md`

**执行**：
1. 输出一句话背景摘要，确认上下文
2. 请用户描述或粘贴变更内容（PRD、口头描述均可）
3. 只记录**业务行为的变化**，不涉及技术实现：
   - 新增：从无到有的新功能/新行为
   - 修改：已有功能的行为变化（原行为 → 新行为）
   - 删除：下线/废弃的功能
   - 非功能性：体验、文案、配置等变化
4. 直接写入 `01_changes.md`，然后展示结果，等待修改意见
5. 更新 `_state.md`：Step2 打勾，上次停留更新

**格式**：
```markdown
# 变更点 | {需求名} | {日期}

## 新增
- {变更描述}

## 修改
- {原行为} → {新行为}

## 删除
- {变更描述}

## 非功能性
- {变更描述}
```

> ⚠️ 只描述「做什么」，不描述「怎么做」，技术方案在 Step 4 讨论。
