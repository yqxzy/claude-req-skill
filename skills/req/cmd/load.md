# /req load <id>

**用途**：每次新对话的第一步，确定本次对话操作哪个需求。

**执行**：
1. 在 `~/.claude/requirements/` 下找到 `<id>-*` 目录
2. 读取该目录下的 `_state.md`
3. 输出恢复简报，**第一行必须包含完整路径**（后续命令依赖此信息）：

```
📋 [<id>] <需求名>  路径：~/.claude/requirements/<id>-<名称>/
背景：<背景摘要，一句话>
进度：已完成 Step 1-N，当前在 Step M
选定方案：<方案名> / 未选定
约束：<关键约束，逗号分隔>
上次停留：<_state.md 中的字段>
建议下一步：/req <命令>
```

4. **将路径记住**，本次对话后续所有 `/req` 命令都将操作这个目录
5. 等待用户指令，不主动执行下一步

> 多需求并行时：每个对话各自 `/req load`，互不干扰。
> 同一对话中途切换需求：再次执行 `/req load <other-id>` 即可覆盖。
