# /req solutions — Step 4：方案设计与评估

**加载文件**：`_state.md`、`00_background.md`、`01_changes.md`、`02_codebase.md`

**执行**：
1. 综合三个文件，提出 2～3 个可行方案，每个方案包含：
   - 一句话概述
   - 实现路径（3～5 步）
   - 评估表：可行性 / 开发难度（低中高）/ 风险 / 兼容性
   - Top 2 风险点 + 应对措施
2. 输出方案对比矩阵，给出推荐意见
3. **等待用户选定方案**（不要自动选）
4. 用户选定后写入 `03_solutions.md`，同时：
   - 更新 `_state.md`：Step4 打勾，选定方案 = 「方案X - 一句话描述」，上次停留更新
