# /req background — Step 1：背景信息

**加载文件**：`_state.md`

**执行**：
1. 逐项引导（每次问一个，等用户回答再问下一个）：
   - 业务背景：解决什么问题？谁提的需求？
   - 技术背景：系统现状、相关模块的已有能力
   - 上下游依赖：涉及哪些系统 / 团队
   - 关键约束：不可违反的技术或业务限制
   - 成功标准：如何判断做完了、做对了
2. 直接写入 `00_background.md`（格式：各项标题 + 内容，带日期），然后展示内容等待修改意见
4. 更新 `_state.md`：
   - Step1 打勾
   - 背景摘要 = 业务背景的一句话概括
   - 关键约束 = 逗号分隔
   - 上次停留 = 「Step 1 完成，建议执行 /req changes」
